# Verification Surface

faultline adopts the portable RIPR verification surface with a strict boundary:

- Public badges summarize repo-scoped state.
- PR evidence routes diff-scoped review and follow-up work.
- Neither surface is used as the other.

## Public Badges

`badges/ripr.json` is generated by:

```bash
cargo xtask badges
```

Check it with:

```bash
cargo xtask badges --check
```

The badge is intentionally narrow. It does not claim coverage, mutation success,
correctness, release readiness, or PR merge readiness.

faultline does not publish a `ripr+` badge yet because the `ripr+` endpoint
requires repo-scoped test-efficiency proof. Absence of that badge is deliberate,
not a failed refresh.

## PR Evidence

Diff-scoped artifacts live under `target/ripr/`:

```bash
cargo xtask ripr-pr --base origin/main --head HEAD
cargo xtask ripr-review-comments --base origin/main --head HEAD
cargo xtask ripr-pr-summary
cargo xtask ripr-annotations
cargo xtask impacted-evidence
```

Generated artifacts:

| Artifact | Path | Purpose |
| --- | --- | --- |
| PR evidence JSON | `target/ripr/pr/repo-exposure.json` | Machine-readable diff evidence and routing fields. |
| PR evidence Markdown | `target/ripr/pr/repo-exposure.md` | Human-readable evidence packet. |
| PR summary | `target/ripr/pr/summary.md` | Stable first-screen summary for CI and agents. |
| Review guidance JSON | `target/ripr/review/comments.json` | Advisory line-placeable and summary-only guidance. |
| Review guidance Markdown | `target/ripr/review/comments.md` | Human-readable review guidance. |
| Warning annotations | `target/ripr/review/annotations.txt` | Non-blocking GitHub warning commands from `comments[]` only. |
| Impacted evidence | `target/xtask/impacted-evidence/latest.json` | Mutation-routing receipt; does not run mutation. |

`summary_only[]` and `suppressed[]` entries never become annotations. The default
mode is advisory and non-blocking.

## Mutation Routing

`cargo xtask impacted-evidence` routes verification cost. It may request
targeted mutation when labels or RIPR evidence justify it, but it never executes
mutation itself. Full-owner mutation remains an explicit owner action.
